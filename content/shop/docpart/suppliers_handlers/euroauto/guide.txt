Установка composer.
https://getcomposer.org/download/

Команда php -r может меняться, в зависимости от текущего бинда php. Если на хостинге поддерживается несколько версий, то скорее всего на php забиндена последняя версия. Тогда нужно явно указать версию (напр. php5.4 )

00. Получить доступ к консоли сервера.  

01. Перейти в директорию сайта(для локальной установки).

1. Скачиваем установочный файл
php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"

2. Проверка целостности данных
php -r "if (hash_file('sha384', 'composer-setup.php') === '93b54496392c062774670ac18b134c3b3a95e5a5e5c8f1a9f115f203b75bf9a129d5daa8ba6a13e2cc8a1da0806388a8') { echo 'Installer verified'; } else { echo 'Installer corrupt'; unlink('composer-setup.php'); } echo PHP_EOL;"

Если появился текст "Installer verified", можно устанавливать. 
Если Installer corrupt, то скорее всего, хеш неактуален и установочный файл будет удалён. Текущий можно получить здесь https://getcomposer.org/download/

3. Выполнение установки(Локально)
php composer-setup.php

4. Удаление установочных файлов
php -r "unlink('composer-setup.php');"

После установки composer нужно загрузить с git библиотеку emarref/jwt

( Указание бинда php актуально )

1.
php composer.phar require emarref/jwt

После скачивания библиотеки, в папке сайта будет создана папка vendor.

Теперь нужно закинуть скрипты для поставщика ( только common_interface ) и добавить тех.интерфейс в базу( всё как обычно ). 

В проценке отображаются только новые товары в наличии. При необходимости, можно добавить Б/У, они есть в ответе от API.
